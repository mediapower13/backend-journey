### Important Git Commands
```bash
# Initialize repository
git init

# Check status
git status

# Stage changes
git add <file>
git add .

# Commit with message
git commit -m "type: description"

# Branch operations
git branch <branch-name>
git checkout <branch-name>
git checkout -b <new-branch>
git merge <branch-name>

# Remote operations
git remote add origin <url>
git push origin <branch>
git pull origin <branch>

# View history
git log
git log --oneline --graph --all
git diff
git blame <file>

# Undo changes
git revert <commit>  # Creates new commit that undoes changes
git reset <commit>   # Moves HEAD to specified commit
```

## Challenges Faced
- Understanding the difference between `git revert` and `git reset`
- Merge conflict resolution strategies

## Solutions Found
- **git revert**: Safe for shared branches, creates a new commit that undoes changes
- **git reset**: Rewrites history, use only for local branches
  - `--soft`: Keeps changes staged
  - `--mixed`: Keeps changes unstaged (default)
  - `--hard`: Discards all changes
